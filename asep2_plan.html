01
System Overview
Architecture · Working Principle · Block Diagram

VOLTGUARD is a real-time electricity theft detection system built on the ESP32 microcontroller. It continuously monitors supply and load current/voltage, applies fuzzy-logic-style thresholds to identify anomalies, and instantly alerts the grid operator via a web dashboard, SMS, and relay cut-off.

📡
Data Flow
Sensors → ESP32 ADC → Processing → WiFi → Firebase → Dashboard update every 500ms
⚡
Theft Detection
Compares supply vs load current. Difference > 10% triggers alert, relay cut-off, SMS, and dashboard notification.
🛡️
Safety
All AC mains handled through optocoupled and transformer-isolated sensors. ESP32 never touches mains voltage.

Step-by-Step Build Guide
Assembly · Testing · Integration
🔴 STOP! Read Section 10 (Safety) FIRST before touching any AC mains wiring. Build and test the DC side completely before connecting to AC mains. Have a supervisor present when working with mains.
1
Set Up the Development Environment
Install Arduino IDE 2.x or VS Code + PlatformIO. Add ESP32 board support: in Arduino IDE go to File → Preferences → Additional Board URLs and paste:

https://dl.espressif.com/dl/package_esp32_index.json

Then go to Tools → Board → Board Manager, search "esp32" and install the Espressif package. Install these libraries via Library Manager: Firebase ESP32, LiquidCrystal_I2C, ArduinoJson. Select board: DOIT ESP32 DEVKIT V1, port: your COM port.
2
Test the ESP32 First
Before connecting anything, upload the Blink sketch to confirm the board is working. Open File → Examples → 01.Basics → Blink and upload. The onboard LED (GPIO2) should blink every second. If it does, your ESP32 is functional. Test WiFi by uploading a simple WiFi scan sketch and verifying it finds nearby networks.
3
Wire the ACS712 Current Sensors
Power each ACS712 module from ESP32 3.3V and GND. Connect the OUT pin to GPIO35 (supply sensor) and GPIO32 (load sensor). Add a 10kΩ pull-down resistor from each OUT pin to GND. Open Serial Monitor and read raw ADC values — at zero current you should see approximately 2048 (midpoint of 12-bit ADC). Verify: gently touch both IP+ and IP- to a AA battery through a 100Ω resistor and confirm the ADC reading changes.

Important: The current-carrying wire of the circuit passes through the ACS712 module terminals — it does NOT need to be cut; there are two screw terminals (IP+ and IP−) through which the load wire loops.
4
Wire the ZMPT101B Voltage Sensor
Power ZMPT101B from 5V and GND (this module requires 5V). Connect OUT to GPIO34 on ESP32. Initially, test with a safe low-voltage AC source (like a 9V AC transformer secondary). Connect ZMPT101B primary terminals to the AC transformer secondary. Read the ADC and adjust the onboard potentiometer until you see a clean oscillating signal centered around 2048. Once calibrated at low voltage, the mains connection follows the same principle but requires full safety measures (Section 10).
5
Wire the Relay Module
Connect relay module: VCC → 5V, GND → GND, IN → GPIO26 (through the driver transistor). The transistor circuit: ESP32 GPIO26 → 1kΩ resistor → BC547 Base. BC547 Collector → Relay IN pin. BC547 Emitter → GND. Place the 1N4007 flyback diode across the relay coil terminals (cathode to the positive side). Test by running: digitalWrite(26, HIGH) — you should hear a click. Use a multimeter to verify the relay COM-NO contact closes when triggered.
6
Wire the LCD and Buzzer
LCD: Connect I2C LCD backpack: VCC → 5V, GND → GND, SDA → GPIO21, SCL → GPIO22. Upload the I2C scanner sketch to verify address (usually 0x27 or 0x3F). Adjust LCD contrast via backpack potentiometer.

Buzzer: Active buzzer through BC547: GPIO27 → 1kΩ → BC547 Base, Collector → Buzzer+ (5V side), Emitter → GND. GND pin of buzzer also to GND. Test with a quick HIGH/LOW pulse.
7
Set Up HLK-PM01 Power Supply (AC Mains — Supervised)
Only do this step with a supervisor and with the circuit disconnected from ESP32 first.

Use screw terminal blocks for all mains connections. Connect HLK-PM01 Input L and N to mains L and N. Connect HLK OUT+ to ESP32 VIN, OUT− to GND. Power on and measure HLK output with a multimeter — should be 5.0V ±0.25V. Then disconnect mains, reconnect all DC components to ESP32, and power on again. Your entire system should run from mains power now.
8
Full System Integration Test
Upload the complete firmware (Section 6). Connect to your WiFi network. Open Serial Monitor at 115200 baud — you should see voltage, supply current, load current, power, and theft status printing every 500ms. Verify Firebase receives the data (Section 7). Simulate theft by slightly disconnecting the load-side ACS712 sensor wire to create an artificial imbalance — the buzzer should sound, relay should click, and the dashboard should show a theft alert.
05
Theft Detection Logic
Mathematical Model · Conditions · Decision Tree

Electricity theft is detected by comparing the current entering a distribution segment (supply current) with the current actually consumed by registered loads (load current). Any significant difference indicates either theft or an illegal bypass.

📐 Sensor Reading Formulas
// ACS712-30A: Sensitivity = 66 mV/A, Offset = 2.5V (5V supply) → 1.65V (3.3V supply)
ADC_raw = analogRead(pin) // 0–4095 for ESP32 12-bit V_adc = ADC_raw × (3.3 / 4095) // Convert to voltage // Peak current (measured over many samples): I_peak = (V_adc − V_offset) / Sensitivity = (V_adc − 1.65) / 0.066 // in Amperes // RMS current (sample N points over one AC cycle ~20ms @ 50Hz): I_rms = √( (1/N) × Σ(I_sample²) ) // ZMPT101B voltage (calibration constant K found experimentally): V_rms = K × √( (1/N) × Σ(V_sample²) ) // K ≈ 210–240 (calibrate!) // Power: P_real = V_rms × I_rms × Power_Factor // Watts (PF ≈ 0.85 assumed)
🔍 Theft Detection Conditions
⚡
Condition 1 — Current Imbalance
Primary detection method.
Imbalance% = |I_supply − I_load| / I_supply × 100 // Normal: Imbalance% < 5% (line losses) // Suspect: Imbalance% 5–10% (investigate) // THEFT: Imbalance% > 10% → ALERT!
A significant difference between supply and load current means current is going somewhere unaccounted — an illegal bypass.
📉
Condition 2 — Voltage Drop
Secondary indicator.
V_drop% = |V_nominal − V_measured| / V_nominal × 100 // Normal: V_drop% < 5% // Suspect: V_drop% 5–10% // Abnormal: V_drop% > 10% → flag
Sudden unexplained voltage drop at the meter indicates excess current being drawn through an illegal Sudden unexplained voltage drop at the meter indicates excess current being drawn through an illegal connection.
📊
Condition 3 — Sudden Spike
Transient detection.
Δ I = |I_current − I_previous| // If ΔI > 5A in < 100ms → suspicious spike // Combine with imbalance check for confirmation
A sudden unexplained jump in supply current with no corresponding load increase signals an illegal hook-up event.
🧠
Combined Decision Logic
THEFT_DETECTED = ( Imbalance% > THRESH_IMBALANCE // 10% AND V_drop% > THRESH_VOLTAGE // 5% ) OR ( Imbalance% > THRESH_CRITICAL // 25% ) // Any single extreme OR both moderate → alert
Using two conditions reduces false positives from momentary load switching.

// ── State Variables ───────────────────────────────────
struct PowerReadings {
  float voltageRMS;
  float supplyCurrentRMS;
  float loadCurrentRMS;
  float realPower;
  float imbalancePercent;
  float voltageDrop;
  bool  theftDetected;
  String status;   // "NORMAL", "WARNING", "THEFT"
  unsigned long timestamp;
};

PowerReadings readings;
int   theftCounter     = 0;
bool  relayTripped     = false;
bool  smsSent          = false;
unsigned long lastFirebaseSend = 0;

// ═══════════════════════════════════════════════════
// SENSOR READING — RMS Calculation
// ═══════════════════════════════════════════════════
float readCurrentRMS(int pin) {
  double sumSquares = 0.0;
  int validSamples = 0;

  for (int i = 0; i < SAMPLE_COUNT; i++) {
    int   raw    = analogRead(pin);
    float vADC   = (raw / ADC_RESOLUTION) * ADC_VREF;
    float iInst  = (vADC - ACS712_OFFSET_V) / ACS712_SENSITIVITY;

    // Noise floor filter — ignore tiny readings
    if (abs(iInst) > 0.05f) {
      sumSquares += (iInst * iInst);
      validSamples++;
    }
    delayMicroseconds(SAMPLE_INTERVAL_US);
  }

  if (validSamples == 0) return 0.0f;
  return sqrt(sumSquares / validSamples);
}

float readVoltageRMS() {
  double sumSquares = 0.0;

  for (int i = 0; i < SAMPLE_COUNT; i++) {
    int   raw   = analogRead(PIN_VOLTAGE);
    float vADC  = (raw / ADC_RESOLUTION) * ADC_VREF;
    float vInst = (vADC - (ADC_VREF / 2.0f));  // Remove DC offset
    sumSquares += (vInst * vInst);
    delayMicroseconds(SAMPLE_INTERVAL_US);
  }

  float vRMSraw = sqrt(sumSquares / SAMPLE_COUNT);
  return vRMSraw * ZMPT_VCAL;  // Apply calibration constant
}

// ═══════════════════════════════════════════════════
// THEFT DETECTION ALGORITHM
// ═══════════════════════════════════════════════════
void detectTheft() {
  // Calculate current imbalance percentage
  if (readings.supplyCurrentRMS < 0.1f) {
    readings.imbalancePercent = 0.0f;
    readings.status = "NORMAL";
    return;
  }

  readings.imbalancePercent =
    (abs(readings.supplyCurrentRMS - readings.loadCurrentRMS)
     / readings.supplyCurrentRMS) * 100.0f;

  readings.voltageDrop =
    (abs(V_NOMINAL - readings.voltageRMS) / V_NOMINAL) * 100.0f;

  // ── Decision logic ───────────────────────────────
  bool imbalanceTheft  = readings.imbalancePercent > THRESH_IMBALANCE_THEFT;
  bool imbalanceCrit   = readings.imbalancePercent > THRESH_IMBALANCE_CRIT;
  bool imbalanceWarn   = readings.imbalancePercent > THRESH_IMBALANCE_WARN;
  bool voltageLow      = readings.voltageDrop      > THRESH_VOLTAGE_WARN;

  if (imbalanceCrit || (imbalanceTheft && voltageLow)) {
    theftCounter++;
    if (theftCounter >= ALERT_DEBOUNCE_COUNT) {
      readings.theftDetected = true;
      readings.status = "THEFT";
      triggerAlert();
    }
  } else if (imbalanceWarn) {
    theftCounter = max(0, theftCounter - 1);
    readings.theftDetected = false;
    readings.status = "WARNING";
  } else {
    theftCounter = 0;
    readings.theftDetected = false;
    readings.status = "NORMAL";
    // Reset relay and buzzer if system normalizes
    if (relayTripped) resetRelay();
  }
}

// ═══════════════════════════════════════════════════
// ALERT ACTIONS
// ═══════════════════════════════════════════════════
void triggerAlert() {
  Serial.println("[ALERT] ⚡ ELECTRICITY THEFT DETECTED!");

  // 1. Trip relay (cut power)
  if (!relayTripped) {
    digitalWrite(PIN_RELAY, LOW);   // Most relay modules are active LOW
    relayTripped = true;
    Serial.println("[RELAY] Power cut-off ACTIVATED");
  }

  // 2. Sound buzzer (3 fast beeps)
  for (int i = 0; i < 3; i++) {
    digitalWrite(PIN_BUZZER, HIGH);
    delay(200);
    digitalWrite(PIN_BUZZER, LOW);
    delay(100);
  }

  // 3. Send SMS (only once per theft event)
  if (!smsSent) {
    sendSMSAlert();
    smsSent = true;
  }

  // 4. Blink LED rapidly
  digitalWrite(PIN_STATUS_LED, HIGH);
}

void resetRelay() {
  digitalWrite(PIN_RELAY, HIGH);  // Deactivate relay
  relayTripped = false;
  smsSent      = false;
  Serial.println("[RELAY] Power restored — system normalized");
}

void sendToFirebase() {
  if (!Firebase.ready()) return;

  FirebaseJson json;
  json.set("voltage",         readings.voltageRMS);
  json.set("supplyCurrentA",  readings.supplyCurrentRMS);
  json.set("loadCurrentA",    readings.loadCurrentRMS);
  json.set("powerW",          readings.realPower);
  json.set("imbalancePct",    readings.imbalancePercent);
  json.set("voltageDropPct",  readings.voltageDrop);
  json.set("theftDetected",   readings.theftDetected);
  json.set("status",          readings.status);
  json.set("relayTripped",    relayTripped);
  json.set("timestamp",       (int)(millis() / 1000));

  if (Firebase.setJSON(fbData, "/voltguard/live", json)) {
    Serial.println("[FB] Data sent OK");
  } else {
    Serial.println("[FB] Error: " + fbData.errorReason());
  }

  // Also append to history log (last 100 entries)
  Firebase.pushJSON(fbData, "/voltguard/history", json);
}

void updateLCD() {
  lcd.clear();
  switch (lcdPage % 3) {
    case 0:  // Page 1: Voltage + Supply Current
      lcd.setCursor(0, 0); lcd.print("V:");
      lcd.print(readings.voltageRMS, 1); lcd.print("V ");
      lcd.print(readings.status);
      lcd.setCursor(0, 1); lcd.print("Is:");
      lcd.print(readings.supplyCurrentRMS, 2); lcd.print("A");
      break;
    case 1:  // Page 2: Load Current + Power
      lcd.setCursor(0, 0); lcd.print("IL:");
      lcd.print(readings.loadCurrentRMS, 2); lcd.print("A");
      lcd.setCursor(0, 1); lcd.print("P:");
      lcd.print(readings.realPower, 1); lcd.print("W");
      break;
    case 2:  // Page 3: Imbalance + Status
      lcd.setCursor(0, 0); lcd.print("Imbal:");
      lcd.print(readings.imbalancePercent, 1); lcd.print("%");
      lcd.setCursor(0, 1);
      if (readings.theftDetected) { lcd.print("!! THEFT ALERT !!"); }
      else { lcd.print("System OK"); }
      break;
  }
  lcdPage++;
}

// SETUP
// ═══════════════════════════════════════════════════
void setup() {
  Serial.begin(115200);
  gsmSerial.begin(GSM_BAUD, SERIAL_8N1, GSM_RX, GSM_TX);

  // GPIO setup
  pinMode(PIN_RELAY,      OUTPUT);  digitalWrite(PIN_RELAY, HIGH); // Relay OFF
  pinMode(PIN_BUZZER,     OUTPUT);  digitalWrite(PIN_BUZZER, LOW);
  pinMode(PIN_STATUS_LED, OUTPUT);  digitalWrite(PIN_STATUS_LED, LOW);

  // ADC setup — use 11dB attenuation for 0–3.3V range
  analogSetAttenuation(ADC_11db);
  analogReadResolution(12);

  // LCD init
  lcd.init(); lcd.backlight();
  lcd.setCursor(0, 0); lcd.print("VOLTGUARD v1.0");
  lcd.setCursor(0, 1); lcd.print("Initializing...");

  // WiFi connection
  Serial.print("[WiFi] Connecting to "); Serial.println(WIFI_SSID);
  WiFi.begin(WIFI_SSID, WIFI_PASSWORD);
  int wifiAttempts = 0;
  while (WiFi.status() != WL_CONNECTED && wifiAttempts < 20) {
    delay(500); Serial.print("."); wifiAttempts++;
  }
  if (WiFi.status() == WL_CONNECTED) {
    Serial.println("\n[WiFi] Connected! IP: " + WiFi.localIP().toString());
    lcd.setCursor(0, 1); lcd.print("WiFi OK!        ");
  } else {
    Serial.println("\n[WiFi] FAILED — offline mode");
    lcd.setCursor(0, 1); lcd.print("WiFi FAILED     ");
  }

  // Firebase init
  fbConfig.host             = FIREBASE_HOST;
  fbConfig.signer.tokens.legacy_token = FIREBASE_AUTH;
  Firebase.begin(&fbConfig, &fbAuth);
  Firebase.reconnectWiFi(true);
  fbData.setResponseSize(1024);

  delay(1500);
  Serial.println("[SYSTEM] VOLTGUARD ready!");
  lcd.clear(); lcd.print("System Ready!");
}

// ═══════════════════════════════════════════════════
// MAIN LOOP
// ═══════════════════════════════════════════════════
void loop() {
  unsigned long now = millis();

  // ── 1. Read all sensors ─────────────────────────
  readings.voltageRMS        = readVoltageRMS();
  readings.supplyCurrentRMS  = readCurrentRMS(PIN_CURRENT_SUP);
  readings.loadCurrentRMS    = readCurrentRMS(PIN_CURRENT_LOAD);
  readings.realPower         = readings.voltageRMS
                               * readings.supplyCurrentRMS
                               * 0.85f;  // assumed PF
  readings.timestamp         = now;

  // ── 2. Run theft detection ───────────────────────
  detectTheft();

  // ── 3. Serial debug output ───────────────────────
  Serial.printf("V=%.1fV | Is=%.2fA | Il=%.2fA | P=%.1fW | Imbal=%.1f%% | %s\n",
    readings.voltageRMS, readings.supplyCurrentRMS,
    readings.loadCurrentRMS, readings.realPower,
    readings.imbalancePercent, readings.status.c_str());

  // ── 4. Send to Firebase every 2 seconds ─────────
  if (now - lastFirebaseSend >= FIREBASE_SEND_INTERVAL) {
    sendToFirebase();
    lastFirebaseSend = now;
  }

  // ── 5. Update LCD every 3 seconds ───────────────
  if (now - lastLCDUpdate >= 3000) {
    updateLCD();
    lastLCDUpdate = now;
  }

  // ── 6. Status LED heartbeat ──────────────────────
  if (!readings.theftDetected) {
    // Normal: slow blink every 1 second
    digitalWrite(PIN_STATUS_LED, (now / 1000) % 2);
  }

  delay(50);  // Small delay for stability
}

  const liveRef = ref(db, 'voltguard/live');

  onValue(liveRef, (snapshot) => {
    const data = snapshot.val();
    if (data) callback(data);
  });

// ── Alerts Listener ────────────────────────────────
export function subscribeAlerts(callback) {
  const alertRef = ref(db, 'voltguard/alerts');
  onValue(alertRef, (snapshot) => {
    const alerts = [];
    snapshot.forEach(child => alerts.push({ id: child.key, ...child.val() }));
    callback(alerts.reverse());   // newest first
  });
}

// ── History Query (last 50 readings) ──────────────
import { query, limitToLast, orderByKey } from 'firebase/database';

export function subscribeHistory(callback) {
  const histRef = query(
    ref(db, 'voltguard/history'),
    orderByKey(), limitToLast(50)
  );
  onValue(histRef, (snapshot) => {
    const history = [];
    snapshot.forEach(child => history.push(child.val()));
    callback(history);
  });
}

// ── Acknowledge Alert ─────────────────────────────
import { update } from 'firebase/database';
export function resolveAlert(alertId) {
  update(ref(db, `voltguard/alerts/${alertId}`), { resolved: true });
}

import { useState, useEffect } from 'react';

// Drop-in hook — use in your VOLTEX dashboard component
export function useVoltguardData() {
  const [liveData,  setLiveData]  = useState(null);
  const [alerts,    setAlerts]    = useState([]);

    subscribeHistory(setHistory);

  // Derived values for dashboard cards
  const derived = liveData ? {
    voltage:       liveData.voltage?.toFixed(1)      ?? '--',
    supplyA:       liveData.supplyCurrentA?.toFixed(2) ?? '--',
    loadA:         liveData.loadCurrentA?.toFixed(2)   ?? '--',
    powerW:        liveData.powerW?.toFixed(0)         ?? '--',
    imbalancePct:  liveData.imbalancePct?.toFixed(1)   ?? '--',
    theftProb:     Math.min(100, liveData.imbalancePct * 4),
    status:        liveData.status ?? 'UNKNOWN',
    theftDetected: liveData.theftDetected ?? false,
    relayTripped:  liveData.relayTripped  ?? false,
    efficiency:    liveData.supplyCurrentA > 0

      load:    h.loadCurrentA,
      voltage: h.voltage,
      prob:    Math.min(100, h.imbalancePct * 4)
    }))
  } : {};

  return { liveData, derived, alerts, history, connected };
}

Advanced Features
Relay Cut-Off · SMS Alerts · OTA · Watchdog
⚡
Relay Auto Cut-Off
When theft is confirmed (debounce = 3 consecutive detections), GPIO26 goes LOW, energizing the relay and breaking the NC contact. The load circuit opens instantly. The relay only resets when the operator clicks "Restore Power" on the dashboard — preventing re-connection without human acknowledgement.
Fail-safe design: Relay is normally-closed (NC). A power failure to ESP32 will open the circuit — preventing theft during a power outage to the controller.
📱
SMS Alert via SIM800L
SIM800L connects via UART2 (GPIO16/17). On theft detection the ESP32 sends AT commands to compose and send an SMS containing: voltage, supply current, load current, imbalance %, and timestamp. The SIM800L needs a separate 3.7–4.2V supply (Li-ion cell or dedicated regulator). A 1000µF capacitor on its power pin prevents brownouts during GSM transmission peaks (2A peak).
🔄

Hardware Watchdog
ESP32 has a built-in watchdog timer. Enable it to auto-reset if the firmware hangs:
C++
#include <esp_task_wdt.h>
// In setup():
esp_task_wdt_init(30, true); // 30s timeout

If the main loop takes more than 30 seconds (WiFi hang, etc.), the ESP32 automatically resets.

Safety Guidelines
Mains Isolation · Protection · Lab Precautions
☠️ 230V AC mains can kill. Even a brief contact with a live wire at mains voltage can cause cardiac arrest. Read every rule below before touching any AC wiring. Have a knowledgeable person present at all times when working with mains power.
🔴 Mandatory Safety Rules
1
Always power off before wiring. Disconnect the mains plug / flip the breaker before making any changes to AC-side wiring. Verify with a non-contact voltage tester.
2
Never use bare-end jumper wires on AC mains. Use only proper screw terminal blocks (KF301 type) with insulated crimp ferrules on wire ends for mains connections.
3
Use an isolation transformer during development. A 1:1 isolation transformer between the mains outlet and your project prevents electrocution by removing earth-referenced hazard. Available at electronics stores for ~₹500.
4
Completely separate AC and DC wiring. Route AC wires (L, N) on one side of the enclosure. DC wires (3.3V, 5V, signals) on the other. Use different wire colours: Red/Black for AC, different colours for DC.
5
Enclose all AC connections. Mount the HLK module, relay, and all AC terminal blocks inside a plastic (not metal) enclosure with proper cover. Label "DANGER — HIGH VOLTAGE" on the outside.
6
Use a 2A fuse on the Live (L) wire. Add an in-line fuse holder with a 2A fuse in series with the Live wire before the HLK module. This is your primary overcurrent protection.
7
One hand rule. When probing or measuring near any powered AC circuit, keep one hand behind your back. This prevents a current path across your heart if accidental contact occurs.
8
Ground the enclosure if using a metal box. Connect the metal chassis to earth ground via a green/yellow wire. For a college demo, a plastic enclosure avoids this issue entirely.
9
Verify isolation between AC and DC with a multimeter. Before powering on, check resistance between any ESP32 pin and the mains Live wire — it must read OL (open, infinite). Any continuity means a dangerous fault.
10
Never work alone. Always have a second person present who can call for help and knows to disconnect power if something goes wrong. They should not touch anything themselves — just be able to cut power at the breaker.
🛡️ Isolation Architecture Summary
🔌
HLK-PM01 Module
Built-in reinforced insulation between primary (230V) and secondary (5V). Rated for 3kV isolation. Certified for mains use. The DC output has no electrical connection to the mains input.
🔋
ACS712 Isolation
Hall-effect sensor with no galvanic connection between AC current path and signal output. The AC wire passes through a magnetic core — completely isolated from the signal pin.
⚙️
ZMPT101B Isolation
Transformer-based voltage sensor. Primary winding connects to mains, secondary to ESP32 ADC through opamp. Transformer provides galvanic isolation with 4kV isolation rating.
✅ Safe Architecture Summary: With HLK-PM01 + ACS712 + ZMPT101B, the ESP32 and all breadboard/signal electronics are completely electrically isolated from AC mains. No mains voltage can reach the ESP32 under any normal operating condition. The only connection is magnetic (ACS712) and inductive (ZMPT101B).
